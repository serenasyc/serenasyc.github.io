Fitting Operator

Definite clause reasoning with negation as failure.

The Fitting operator was implemented as an assignment for my Knowledge Representation and Reasoning course at SFU.

It carries out a simple, generalized forward-chaining reasoning on a set of clauses. The clauses are similar to definite clauses in that each requires one positive head atom but negation is allowed in the body. As such, contradictions are possible.

The code can be found here.

Optimizing Cost

constraint programming

The Pizza Problem is taken from the the 2015 LP/CP Programming Contest. The problem description can be found here. Basically, you have a set of pizzas, each with a price, and a set of vouchers, each of the form "buy x pizzas, get y free". You want to buy all pizzas for the minimum price using the vouchers at hand.

The problem specification is written for the IDP system, a knowledge base system for an extension of first order logic. It can be run with the IDP Web-IDE.
